# F1 Pit Stop Prediction — Handover for Codex

Kaggle [Playground Series S6E5](https://www.kaggle.com/competitions/playground-series-s6e5). Binary classification: predict whether an F1 driver will pit on the next lap. Metric: AUC-ROC.

## Current state

- **OOF best: 0.953860** — L1 logistic stack of 107 OOF models
- **Predicted LB if submission tracking holds (gap ~0.0008): ~0.953**
- **Submission ready to send: `submissions/best_5h_stack.csv`** — blocked until Kaggle daily limit resets (00:00 UTC)
- Top of LB: 0.95476 (Chris Deotte). Cluster of 12+ teams at exactly 0.95407 = a public notebook fork. Top 1% cutoff: 0.95413.

## The dataset (what to know)

- **439k train rows, 188k test, 16 features**. Synthetic, generated from an original 101k-row dataset (`data/orig/f1_strategy_dataset_v4.csv`) which the organizers explicitly invited us to use.
- **Target labels are NOISY**. `PitNextLap[N]` agrees with `PitStop[N+1]` only 81% of the time. Likely generated by a teacher model probabilistically.
- **Key feature `Normalized_TyreLife` was removed** by the organizers because it makes prediction trivial. We can reconstruct it: `NTL = TyreLife / max(TyreLife in (Driver, Race, Year, Stint))`. But the reconstruction has bias (median diff 0.13 from true NTL) because **fake driver codes are randomized per row, not per session** — so grouping by fake Driver corrupts the stint max.
- **28% of competition rows have an EXACT key match to original** on (Race, Year, LapNumber, Stint, TyreLife, Position, Compound, PitStop). For these rows we can read TRUE NTL.
- **Train/test is lap-level random within sessions**: 96% of (Race, Year, Driver) sessions span both. So random k-fold CV is correct (NOT GroupKFold).
- **OOF↔LB tracking gap is stable at ~0.0008** (OOF slightly optimistic). Local OOF improvements >0.001 should translate to LB.

## What worked (in order of impact)

1. **Drop the `Driver` feature** (+0.005 LB). 887 fake codes that the model overfits per-fold. Single biggest move of the project.
2. **Train on (comp + orig) combined** (+0.001 LB). Add the 101k orig rows to every fold's training set. CV folds stay on comp rows only. This is "v15".
3. **Logit-space ensembling** of v15 with v10 (multi-resolution orig aggregates as features). Best fixed blend: v15(3) + v10(1) → LB 0.95213.
4. **L1-logistic stacking on 107 OOFs** (+0.0001 OOF over manual blend). C=0.05, saga solver, drops ~30% of redundant LGBM seeds.
5. **Multi-class diversity**: RealMLP from PyTabKit gives correlation 0.91 with LGBM (vs 0.99 LGBM-LGBM). Even at OOF 0.945 standalone, it adds 0.000035 to the stack.

## What didn't work

- `Driver` as a feature, anything `Driver_*` interaction (memorizes per-fold)
- Reconstructed NTL alone (biased; needs to be combined with true NTL via `coalesce`)
- Augmentation reverse-engineering / NN matching to orig (perturbation noise too large)
- Look-ahead leakage via `PitStop[N+1]` (only 41% test coverage, 81% noisy when found)
- Lag features (e.g., `LapTime[N-1]`) — corrupted because previous "lap" of a fake driver is a random other driver
- Pseudo-labeling (threshold 0.92/0.05): added ~nothing
- Adding fastf1 data as training rows: derived features (`Cumulative_Degradation`, etc.) don't align with orig schema; v16 was slightly *worse* than v15
- Sample-weighting matched rows higher (3×): didn't help
- Forward-selection stacking: too slow, equal-weight or L1 wins
- TabR (`RealTabR_D_Classifier`): API mismatch, didn't get it working

## To push toward 0.954+ (top 5-10%)

Ranked by expected lift:

### 1. Fix the fastf1 data integration (RANK 1, predicted +0.001-0.003)
The fastf1 fetch worked (67k rows from real F1 telemetry) but the conversion to orig schema is wrong. `Cumulative_Degradation`, `LapTime_Delta`, and the `PitNextLap` derivation in `fastf1_to_orig.py` don't match how orig computes them. Fix by:
- Reverse-engineer how orig computes `Cumulative_Degradation` (it's not just cumulative LapTime_Delta — there's some normalization)
- Verify `PitNextLap` semantics: in orig, it spans ~10 consecutive laps before a pit (a window flag), not just one lap before pit. The fastf1 conversion uses one-lap shift which is wrong.
- Once aligned, training v15 + fastf1 should give a ~+0.001 OOF lift since it adds ~50k clean rows.

### 2. More RealMLP seeds (RANK 2, predicted +0.0001-0.0005 OOF, possibly more on LB)
RealMLP corr with LGBM is 0.91 (vs 0.99 LGBM-LGBM). One seed already added +0.000035 to the stack. 5+ seeds averaged should compound.
- `python src/v15_realmlp.py SEED` — takes ~70 min per seed on CPU.
- Also try `RealMLP_HPO_Classifier` (PyTabKit's tuned variant) — likely stronger single AUC.

### 3. Logistic stacking refinements (RANK 3, predicted +0.0001-0.0003)
Current L1 logistic at C=0.05 on 107 OOFs gives 0.9539. Try:
- ElasticNet (mix of L1+L2) — sklearn `LogisticRegression(penalty='elasticnet', l1_ratio=0.5, solver='saga')`
- Stacked stacking: train another logistic on (model OOFs + L1 stack OOF) — might capture residual signal
- Bayesian model averaging using validation log-loss as weights

### 4. Re-investigate the 0.954 cluster (RANK 4, possibly the breakthrough)
The cluster at exactly 0.95407 is 12+ teams using one specific public notebook. Ravi Ramakrishnan's `PlaygroundS6E5|Public|Baseline|V1` (30 votes) and Imrankhan's `PS|S6|E5:0.95130` (34 votes) are candidates. The 0.95130 notebook does CatBoost+LGBM+XGB stacking with logistic level-2 stacking (similar to our approach, no orig data). The 0.95407 notebook may be doing something specific we haven't seen — pull it via `kaggle kernels pull` and check.

### 5. Don't pursue (negative findings, save time)
- **Driver disambiguation**: dead. Per-session fake-driver→real-driver purity = 0.33 (random for 20 drivers). Same fake driver appears in 30-100 sessions but maps to different real drivers each time.
- **Augmentation reverse-engineering**: dead. Perturbation std on LapTime is ~32, too noisy for parent identification.
- **Lag features within session**: hurts OOF because previous "lap" of a fake driver is from a random real driver.
- **More LGBM seeds**: saturated. 50+ seeds tried; each adds 0.00001-0.00005.
- **Cat-interaction features** (`Driver_Race`, `Race_Year` etc.): destroys per-fold calibration even if per-fold AUC looks fine.

## Final stacking script (current best)

```python
import numpy as np, pandas as pd, glob
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_auc_score
from sklearn.model_selection import StratifiedKFold
from scipy.special import logit, expit

y = pd.read_csv('data/train.csv').sort_values('id')['PitNextLap'].astype(int).values
oofs = {}
for f in sorted(glob.glob('submissions/*_oof.npy')):
    name = f.split('/')[-1].replace('_oof.npy', '')
    arr = np.load(f)
    if len(arr) == len(y) and not np.isnan(arr).any() and roc_auc_score(y, arr) > 0.85:
        oofs[name] = arr

X = np.stack([logit(np.clip(oofs[n], 1e-7, 1-1e-7)) for n in sorted(oofs.keys())], axis=1)
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
meta = np.zeros(len(y))
for fold, (tr, va) in enumerate(skf.split(X, y)):
    lr = LogisticRegression(C=0.05, max_iter=200, solver='saga', penalty='l1')
    lr.fit(X[tr], y[tr])
    meta[va] = lr.predict_proba(X[va])[:, 1]
print(f'L1 stack OOF: {roc_auc_score(y, meta):.6f}')

# Build submission
test_preds = {n: np.load(f'submissions/{n}_test.npy') for n in oofs}
X_test = np.stack([logit(np.clip(test_preds[n], 1e-7, 1-1e-7)) for n in sorted(oofs.keys())], axis=1)
lr_final = LogisticRegression(C=0.05, max_iter=200, solver='saga', penalty='l1').fit(X, y)
test_meta = lr_final.predict_proba(X_test)[:, 1]
ids = pd.read_csv('data/test.csv').sort_values('id')['id'].values
pd.DataFrame({'id': ids, 'PitNextLap': test_meta}).to_csv('submissions/best.csv', index=False)
```
